## 1. 背景与目标

CloudForge 需要为每个企业租户提供一套独立的组织通讯录。企业可以先录入尚未注册平台账号的员工，维护部门和任职关系；员工注册后再与已有 Employee 关联。企业外部协作者使用独立的 External Contact 身份，只能访问明确分享给自己的业务资源，不能进入内部通讯录。

- 让每个企业拥有清晰、独立的组织和员工信息。
- 统一 Employee、External Contact 和外部协作的产品边界。
- 支持员工录入、批量导入、邮箱关联、组织浏览和搜索。
- 员工离职、停用或外部联系人停用后，立即阻止其继续进入对应企业。

## 2. 范围与非目标

### 2.1 范围

- 组织树的创建、调整、排序和停用。
- Employee 的手动录入、修改、停用、离职和恢复。
- Employee 的主部门和辅助任职。
- 通过已确认工作邮箱或邀请链接关联 User。
- 全企业通讯录浏览和搜索。
- 租户超级管理员角色。
- External Contact 的建立、查询、停用和重新启用。
- 长期有效的业务资源协作授权。
- Employee 批量导入、预览和结果反馈。
- 组织及人员变更记录。

### 2.2 非目标

- 对接第三方人力资源系统、钉钉、飞书或企业微信。
- 私人手机号、私人邮箱、证件、住址、薪资等敏感人事数据。
- 保密组织、敏感字段和额外授权规则。
- 一个组织节点同时属于多个上级。
- External Contact 授权到期、续期和到期提醒。
- 自定义租户角色和复杂权限编排。
- 匿名外部访问。
- 对外开放平台和第三方应用接入。

## 6. 核心流程

### 6.1 手动录入与邮箱关联

1. 租户超级管理员录入 Employee 的工作信息和组织任职。
2. Employee 可以在尚未关联 User 时进入通讯录。
3. 员工使用工作邮箱注册并完成邮箱确认。
4. 只有邮箱唯一匹配且 Employee 尚未关联时，系统才自动完成关联。
5. 没有匹配、重复匹配或已经关联时停止自动处理，并给管理员明确提示。

### 6.2 邀请加入

1. 租户超级管理员向未关联 User 的 Employee 发出邀请。
2. 邀请只发送到 Employee 的工作邮箱，并且一次性、短时有效。
3. 接受者登录并确认目标邮箱后，完成 User 与 Employee 关联。
4. 邀请已使用、过期、邮箱不匹配或 Employee 已关联时，系统拒绝操作并说明原因。

### 6.3 浏览通讯录

1. Employee 选择一个企业进入通讯录。
2. 系统展示企业组织树和正常 Employee 的工作名片。
3. 搜索和列表默认不显示已离职 Employee。
4. External Contact 不能浏览、搜索或推断内部组织和 Employee。

### 6.4 外部协作

1. 有分享权限的 Employee 在业务资源中输入外部联系人邮箱并发出邀请。
2. 对方完成登录并接受邀请后，成为该企业的 External Contact。
3. External Contact 获得该项业务资源的长期协作权限，但不会进入通讯录。
4. 撤销一项协作只影响对应资源；停用 External Contact 则阻止其继续进入该企业的所有资源。

### 6.5 停用、离职与恢复

1. 租户超级管理员可以临时停用 Employee，也可以确认其离职。
2. 状态变更后，该 Employee 立即不能继续进入当前企业。
3. 已离职 Employee 不再出现在正常列表和搜索中，但历史任职和业务记录继续保留。
4. 重新入职时恢复原 Employee，并建立新的任职记录，不创建重复人员档案。

### 6.6 批量导入

1. 租户超级管理员上传员工导入文件。
2. 系统检查必填内容、重复人员、工作邮箱和组织路径。
3. 系统先展示导入预览及逐行问题，不立即修改正式通讯录。
4. 管理员确认后才正式导入。
5. 完成后展示创建、更新、跳过和失败数量，并提供结果明细。

| FR-001 | 租户超级管理员可以创建、重命名、排序、移动和停用 OrgUnit；组织不能移动到自身或下级之下。 | P0 |
| FR-002 | 租户超级管理员可以手动录入和修改 Employee；Employee 可以先于 User 存在，且不直接携带权限。 | P0 |

| FR-005 | 正常 Employee 可以浏览完整组织树和基础工作名片，并按姓名、工作邮箱、工号和组织搜索。 | P0 |
| FR-006 | 每个企业内置一个租户超级管理员角色；没有该角色的正常 Employee 仍可查看通讯录。 | P0 |

| FR-011 | User 的登录状态只证明本人身份；进入企业时仍需确认其 Employee 或 External Contact 当前有效。 | P0 |

## 8. 业务规则

- BR-001：一次操作只能在一个明确选择的当前企业中进行。
- BR-002：选择一个企业不等于有权进入该企业，系统仍需确认有效人员身份。
- BR-003：一个 Employee 只属于一个企业；一个 User 可以在多个企业分别关联 Employee。
- BR-004：一个 Employee 最多关联一个 User；一个 User 在同一企业最多关联一个 Employee。
- BR-005：一个正常 Employee 必须且只能有一个主任职。
- BR-006：External Contact 永远不能成为 OrgUnit 成员，也不能出现在内部通讯录中。
- BR-007：External Contact 身份本身不产生业务资源权限。
- BR-008：Employee 停用或离职、External Contact 停用后，其下一次企业操作必须立即被拒绝。
- BR-009：Employee 工作邮箱可以修改，但不能改变 Employee 的永久身份。
- BR-010：普通 Employee 不能查看或批量获取全企业 External Contact。
- BR-011：一个企业中的 Employee 失效不能影响同一 User 在其他企业中的有效身份。

## 9. 验收标准

- [ ] AC-001：预录入 Employee 与已确认同邮箱 User 唯一匹配时完成关联，不创建重复 Employee。
- [ ] AC-002：User 邮箱未经本人确认时不能自动关联 Employee。
- [ ] AC-003：一个 Employee 始终只有一个正常主任职，冲突操作会被拒绝。
- [ ] AC-004：正常 Employee 可以浏览和搜索本企业通讯录，已离职 Employee 默认不显示。
- [ ] AC-005：External Contact 不能浏览、搜索或推断内部通讯录内容。
- [ ] AC-006：Employee 停用或离职后立即不能进入当前企业，但其他企业访问不受影响。
- [ ] AC-007：External Contact 只能访问明确授权的业务资源，撤销单项协作不影响其他资源。
- [ ] AC-008：停用 External Contact 后，其在当前企业的全部资源访问立即被拒绝。
- [ ] AC-009：导入文件存在重复工作邮箱或无效组织时，系统展示逐行问题且正式通讯录保持不变。
- [ ] AC-010：重复提交相同导入不会产生重复 Employee 或任职。
- [ ] AC-011：没有租户超级管理员角色的正常 Employee 仍可查看通讯录，但不能进行管理操作。
- [ ] AC-012：重新入职恢复原 Employee 和历史记录，并建立新的任职，不创建重复人员。

## 10. 发布与成功指标

- 发布范围：首批企业的租户超级管理员、Employee 和 External Contact。
- 上线条件：组织、Employee、邮箱关联、通讯录、停用离职、External Contact 和批量导入全部通过验收。
- 成功指标：企业能够独立完成组织和 Employee 初始化；员工能够准确找到同事；人员失效后不能继续进入企业。
- 停止发布条件：出现跨企业数据泄露、离职后仍可访问、External Contact 进入内部通讯录或导入破坏已有数据。
