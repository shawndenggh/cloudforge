## 1. 背景与目标

CloudForge 需要为每个自然人提供一个统一的平台 User。User 先完成注册和登录，再根据自己的企业身份进入一个或多个企业租户。这样，同一个人在不同企业中的员工身份可以相互独立，而登录账号始终保持唯一。

- 解决用户缺少统一账号，无法自主注册和登录的问题。
- 支持企业提前录入 Employee，待员工注册后自动完成关联。
- 支持平台管理员在发生安全或治理问题时禁用 User。
- 为后续找回密码、修改邮箱和第三方登录留下扩展空间，但不纳入 MVP。

## 2. 范围与非目标

### 2.1 范围

- 使用邮箱、显示名称和密码注册。
- 通过邮件确认邮箱归属。
- 使用邮箱和密码登录。
- 登出当前设备。
- 查看和修改自己的显示名称、头像、语言和时区。
- 平台管理员查询、禁用和重新启用 User。
- 邮箱确认后，尝试关联企业中已经预先录入的 Employee。
- 系统首次初始化时建立第一个平台管理员。

### 2.2 非目标

- 修改密码、忘记密码和重置密码。
- 修改主邮箱。
- 手机号注册、登录或找回账号。
- GitHub、企业单点登录或其他第三方登录。
- User 主动停用、注销、删除或自行恢复账号。
- 管理企业组织、Employee、External Contact 或租户角色。
- 通过登录状态直接获得任何企业或业务资源权限。

## 6. 核心流程

### 6.1 注册与邮箱确认

1. 访客填写邮箱、显示名称和密码并提交注册。
2. 系统提示用户前往邮箱完成确认。
3. 用户打开有效的确认链接后，账号变为正常状态。
4. 系统尝试使用已确认邮箱关联企业中尚未绑定的 Employee。
5. 没有匹配或出现冲突时，不影响 User 激活，后续由企业管理员处理。

### 6.2 登录

1. User 输入邮箱和密码。
2. 系统确认邮箱、密码和 User 状态均有效。
3. 登录成功后进入平台。
4. 没有任何可进入企业的 User 仍可登录，但页面提示“暂无可访问企业”。

### 6.3 登出

1. User 在当前设备选择登出。
2. 系统结束当前设备的登录状态并返回登录页面。
3. 其他设备上的登录状态不受影响。

### 6.4 禁用与重新启用

1. 平台管理员找到目标 User 并确认禁用。
2. User 立即退出所有已登录设备，并且不能再次登录。
3. User 不能自行恢复。
4. 平台管理员重新启用后，User 可以再次登录，但原来的登录状态不会恢复。

## 8. 业务规则

- BR-001：每个 User 都有一个永久且不可变的唯一标识，主邮箱不是永久身份。
- BR-002：MVP 每个 User 只有一个主邮箱，忽略大小写和首尾空格后在全平台唯一。
- BR-003：企业中录入的 Employee 工作邮箱不能直接视为 User 已经确认的邮箱。
- BR-004：User 有“未启用、待确认、正常、已禁用”四种状态。
- BR-005：公开注册后的 User 进入“待确认”；完成邮箱确认后进入“正常”。
- BR-006：“未启用”用于系统预创建或历史数据迁移的 User，不能直接登录。
- BR-007：只有“正常”User 可以登录。
- BR-008：只有平台管理员可以禁用或重新启用 User；已禁用 User 不能自行恢复。
- BR-009：登录成功只能证明 User 身份，不能证明其属于某个企业或拥有某项业务权限。
- BR-010：User 在一个企业离职或停用，不改变其全局账号状态，也不影响其他企业中的有效身份。
- BR-011：平台管理员与租户超级管理员是两个独立角色，不能互相推导。
- BR-012：最后一个有效平台管理员不能被禁用，平台管理员也不能禁用自己。

## 9. 验收标准

- [ ] AC-001：新邮箱注册后只产生一个待确认 User，完成邮箱确认前不能登录。
- [ ] AC-002：有效确认链接可以激活 User；同一链接重复使用不会重复改变账号状态。
- [ ] AC-003：正确邮箱和密码可以登录；错误密码和未注册邮箱得到相同的失败提示。
- [ ] AC-004：登出当前设备后，该设备不能继续访问；其他设备不受影响。
- [ ] AC-005：没有关联任何企业的正常 User 可以登录，但不能查看任何企业数据。
- [ ] AC-006：User 修改基础资料后，所有已登录设备及时显示最新资料。
- [ ] AC-007：平台管理员禁用 User 后，该 User 的所有设备立即退出，并且不能再次登录。
- [ ] AC-008：重新启用 User 后可以重新登录，但原来的登录状态不会恢复。
- [ ] AC-009：Employee 关联没有匹配或发生冲突时，User 仍能完成激活并正常登录。
- [ ] AC-010：重复执行系统初始化不会产生重复平台管理员，也不会覆盖已有账号资料和状态。

## 10. 发布与成功指标

- 发布范围：MVP 浏览器用户和平台管理员。
- 上线条件：注册、邮箱确认、登录、登出、禁用、重新启用和 Employee 关联全部通过验收。
- 成功指标：用户可以顺利完成注册和登录；登出及禁用及时生效；没有账号混淆或越权访问。
- 停止发布条件：出现错误 User 登录、禁用后仍可使用、平台管理员越权或用户资料泄露。
