## 1. 背景与目标

CloudForge 当前没有可供产品客户端使用的用户注册、登录和登录态鉴权能力，无法可靠识别当前 User，也无法阻止未登录访问。本次交付建立最小可用的全局用户身份入口。

- 背景与问题：任何人都还不能建立 User、登录 CloudForge 或以稳定身份访问受保护功能。
- 产品目标：用户能够自行注册，通过登录邮箱和密码登录，读取当前用户信息，访问受保护功能并退出登录。
- 方案概述：CloudForge 对外提供注册、登录、当前用户信息、登录态鉴权和退出能力。首版只判断是否登录，不引入角色或业务权限。

## 2. 范围与非目标

### 2.1 范围

- 面向所有人开放用户注册。
- 使用全局唯一的登录邮箱和密码注册。
- 注册成功后自动登录。
- 使用登录邮箱和密码登录。
- 同一 User 可以在多个设备同时登录，各设备登录状态相互独立。
- 已登录 User 可以读取自己的只读用户信息。
- 受保护功能只允许已登录 User 访问。
- User 可以退出当前设备的登录状态。
- 对登录中的过量失败尝试进行临时限制。
- 只交付接口能力，不提供用户界面。

### 2.2 非目标

- 邮箱验证以及将登录邮箱作为可信联系方式。
- 忘记密码、重置密码和修改密码。
- 修改登录邮箱、昵称或其他个人资料。
- 用户注销、停用和用户管理。
- 平台管理员、角色、权限配置和差异化访问能力。
- Tenant、Employee、External Contact 和业务资源授权。
- 用户名、手机号、第三方身份或无密码登录。
- “记住我”、全部设备退出和登录设备管理。
- 注册页、登录页或其他用户界面。

## 4. 用户与权限

| 用户类型 | 目标 | 默认可见范围 | 管理能力 |
| --- | --- | --- | --- |
| 未登录访问者 | 注册或登录 | 注册和登录能力 | 无 |
| 已登录 User | 确认本人身份并使用受保护功能 | 本人的当前用户信息和首版受保护功能 | 退出当前设备 |

首版不区分管理员和普通 User。登录状态相同的 User 对本版受保护功能拥有相同访问能力；这不产生任何 Tenant 或业务资源权限。

## 6. 核心流程

### 6.1 注册并自动登录

1. 访问者提交登录邮箱、密码和确认密码。
2. 系统检查登录邮箱是否已被使用，并检查密码是否符合安全要求。
3. 注册成功后系统立即建立登录状态。
4. 客户端可以直接获取当前用户信息并访问受保护功能。
5. 登录邮箱已被使用、两次密码不一致或密码不符合要求时，系统拒绝注册并说明可纠正的问题。

### 6.2 登录

1. 已注册 User 提交登录邮箱和密码。
2. 凭据正确时系统建立该客户端独立的登录状态。
3. 凭据错误时系统统一提示登录邮箱或密码错误，不透露邮箱是否已注册。
4. 失败尝试过多时，系统临时限制继续尝试，并在限制解除后允许重试。

### 6.3 获取当前用户信息与鉴权

1. 客户端携带当前登录状态请求当前用户信息或其他受保护功能。
2. 登录状态有效时，系统允许访问；当前用户信息只展示用户 ID、登录邮箱和注册时间。
3. 登录状态缺失或失效时，系统拒绝访问，且不返回受保护内容。

### 6.4 退出登录

1. User 在当前设备发起退出。
2. 系统立即结束当前设备的登录状态。
3. 当前设备再次访问受保护功能时被拒绝。
4. 同一 User 在其他设备上的有效登录状态不受影响。

## 7. 功能需求

| 编号 | 功能描述 | 优先级 |
| --- | --- | --- |
| FR-001 | 所有人都可以使用登录邮箱、密码和确认密码注册 User。 | P0 |
| FR-002 | 登录邮箱全局唯一且忽略大小写；大小写不同不能创建重复 User。 | P0 |
| FR-003 | 注册成功后系统自动建立登录状态，User 无需再次登录。 | P0 |
| FR-004 | 已注册 User 可以使用登录邮箱和密码登录。 | P0 |
| FR-005 | 密码至少包含 15 个字符，支持至少 64 个字符和长密码短语，不强制字符类型组合，并拒绝常见或已泄露密码。 | P0 |
| FR-006 | 同一 User 可以同时拥有多个相互独立的登录状态。 | P0 |
| FR-007 | 登录状态连续 12 小时无操作或建立满 7 天后失效，满足任一条件即需重新登录。 | P0 |
| FR-008 | 已登录 User 可以读取只包含用户 ID、登录邮箱和注册时间的当前用户信息。 | P0 |
| FR-009 | 受保护功能必须拒绝未登录访问；首版不对已登录 User 作进一步角色或业务权限区分。 | P0 |
| FR-010 | 退出登录只结束当前设备的登录状态，并立即阻止该设备继续访问受保护功能。 | P0 |
| FR-011 | 登录失败不得透露登录邮箱是否已注册；过量失败尝试必须受到临时限制，且不能造成永久账号锁定。 | P0 |
| FR-012 | 首版只交付可供客户端接入的接口能力，不提供用户界面。 | P0 |

## 8. 业务规则

- BR-001：User 是全局身份，不属于任何一个 Tenant，也不因登录获得 Tenant 或业务资源权限。
- BR-002：登录邮箱比较忽略大小写，同一邮箱只能属于一个 User。
- BR-003：首版不验证登录邮箱归属，因此登录邮箱不能作为可信联系方式、Primary Email 或 Employee 自动关联依据。
- BR-004：符合条件的注册一旦成功，User 立即可用并自动登录。
- BR-005：密码不得因为缺少特定大写字母、数字或特殊字符而被拒绝，但仍须满足长度和弱密码限制。
- BR-006：每次登录建立独立登录状态；一个登录状态失效或退出不影响其他有效登录状态。
- BR-007：登录状态只能证明 User 身份，不能证明管理员身份、Tenant 成员身份或业务访问权。
- BR-008：未登录访问者不得看到任何受保护内容。
- BR-009：登录失败提示必须避免暴露某个邮箱是否已经注册。
- BR-010：失败尝试限制必须能够自动解除，不得将 User 永久锁定。

## 9. 验收标准

- [ ] AC-001：访问者使用未注册邮箱和合规密码注册后立即处于已登录状态，无需再次提交凭据。
- [ ] AC-002：某邮箱注册后，使用仅大小写不同的邮箱再次注册会被拒绝，且不会产生第二个 User。
- [ ] AC-003：少于 15 个字符、常见或已泄露的密码会被拒绝；合规的长密码短语可以注册和登录。
- [ ] AC-004：已注册 User 使用正确的登录邮箱和密码可以登录，使用错误凭据时只看到统一错误提示。
- [ ] AC-005：过量登录失败会触发临时限制；限制解除后可以再次尝试，账号不会被永久锁定。
- [ ] AC-006：登录成功后只能从当前用户信息中获得用户 ID、登录邮箱和注册时间，且不能通过本版能力修改这些信息。
- [ ] AC-007：没有有效登录状态时不能访问受保护功能；具有有效登录状态时可以访问本版受保护功能。
- [ ] AC-008：连续 12 小时无操作或登录满 7 天后，原登录状态不能继续访问受保护功能。
- [ ] AC-009：同一 User 可以在两个设备同时登录；退出其中一个设备后，另一个设备仍保持登录。
- [ ] AC-010：退出后的设备再次获取当前用户信息或访问受保护功能时会被拒绝。
- [ ] AC-011：任何访问者都可以发起注册，注册过程不要求完成邮箱验证。
- [ ] AC-012：本次交付不包含注册、登录或个人信息页面，客户端只通过接口使用这些能力。

## 10. 发布与成功指标

- 发布范围：注册能力面向所有访问者开放，登录和受保护功能面向所有已注册 User 开放。
- 上线条件：注册并自动登录、再次登录、当前用户信息、登录态鉴权、多设备登录、登录状态失效和退出流程全部通过验收。
- 成功指标：新用户无需人工协助即可完成注册并进入登录状态；已注册用户能够再次登录、识别当前身份并可靠退出；未登录访问始终无法获得受保护内容。
- 停止条件：出现重复邮箱创建多个 User、未登录访问受保护内容、退出或超时后仍可访问、密码在任何用户可见反馈中泄露，或正常用户被失败限制永久锁定。
